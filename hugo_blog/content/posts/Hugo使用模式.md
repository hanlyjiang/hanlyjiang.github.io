## 概述
本文用于梳理要如何使用 hugo ，如后续使用中需要使用哪些命令，当前对博客的一些规划，使用规则等；



## 整体布局


支持三个维度：

1. 专题（series）：用于将一个主题相关的内容连接起来；
2. 分类（categories）：主要用于区分文章的类别，当前划分如下几种：
   1. 日游所思：个人感想，日记等等；
   2. 蜻蜓点水：主要是一些轻度技术内容，如博客配置，网络配置，玩机配置等，技术相关的想法，但是不涉及具体技术细节的；
   3. 水漫金山：纯技术相关的内容，如技术学习教程，技术总结等；
3. 标签（tags）：标识文章所涉及的一些关键词，热点，关键技术等等，可以尽量多添加；



使用文件头部 meta 定义的方式来添加对应的维度标记，下面是一个模板：

```TOML
+++
date = '2025-11-01T23:15:13+08:00'
draft = false
title = 'Title'
categories = ["坐岸观水","蜻蜓点水","浮游水面","深入水底",]
tags = ["Hugo", "教程"]
series = ["Blog配置系列"]
+++
```
